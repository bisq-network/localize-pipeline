AI localization for maintainers

Ship translations as normal pull requests.

Localize Pipeline detects changed source strings, translates Java .properties and JSON files, runs quality checks, and opens reviewable PRs from your own CI or server.

Adoption path

Onboard a repository in one reviewable PR.

1

Generate

Run localize bootstrap-pr to create config, glossary, workflow, and a target-repo guide.

2

Validate

Keep dry-run: true, add credentials or a local endpoint, and run preflight checks.

3

Translate

Enable writes, run an initial backfill, and review translations through normal PR review.

Formats and providers

Built for extension without rewriting the pipeline.

Java .properties and JSON are built in. Custom adapters can be loaded through CLI plugins, Action plugin inputs, or Python entry points. AISuite is the default model-provider abstraction, with an explicit OpenAI-compatible fallback for local endpoints.

Operational surface

Keep translation state visible and portable.

localize memory stats --memory-file logs/translation_memory.json localize memory export --memory-file logs/translation_memory.json --output shared-memory.json localize memory import --memory-file logs/translation_memory.json --input shared-memory.json localize memory suggest --source-text "Save change" --locale de --format-id json
